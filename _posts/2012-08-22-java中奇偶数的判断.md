在我的代码编写中，判断一个数字是否奇偶。<br/>
经常的方法是
```
public boolean isOdd(int a){
    if(a%2 == 1){   //是奇数
        return true;
    }
    return false;
}
```
但是这样有一半的时候是错误的。因为这里并没有考虑负数的情况。`System.out.println((-1)%2)`;结果为-1。在上面的方法判断中，负奇数和2取余的结果并不是1.所以上面的判断方法并不科学。正确的方法应该为
```
public boolean isOdd(int a){
    if(a%2 != 0){   //是奇数        
        return true;
    }    
    return false;
}
```
这样的处理能够实现正常的判断。但是性能不是太好。比较优化的方法
```
public boolean isOdd(int a){
    if((a&1) != 1){   //是奇数
        return true;
    }
    return false;
}
```
